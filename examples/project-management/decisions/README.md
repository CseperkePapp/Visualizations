# Decisions

Decision records capture durable choices and their rationale.

## Purpose

Use this folder for decisions that affect project direction, architecture, or repeated workflow behavior.

## Recommended Formats

- Chronological log in `DECISION-LOG.md`
- Standalone records such as `DEC-001-name.md` when a decision needs deeper context

## Decision Record Shape

```markdown
## YYYY-MM-DD: Decision title

**Decision:** One-sentence choice.
**Why:** Rationale and context.
**Alternatives:** Other options considered.
**Impact:** What changes because of this.
**Status:** implemented | in-progress | reverted
```

Use `headers/HEADER-DECISION.md` as the canonical reference for field expectations.

# Workflows

Reusable process recipes for recurring project work.

## Purpose

Use this folder when the same process should be run repeatedly and referenced by many tasks.

## Naming

- Use lower-kebab-case names (for example `release-closeout.md`)
- Keep workflow files unnumbered
- Keep numbered execution records in `tasks/`

## Recommended Structure

Use the canonical workflow header/template:

- `headers/HEADER-WORKFLOW.md`
- `templates/TEMPLATE-WORKFLOW.md`

## Example Categories

- Release closeout
- Documentation drift audit
- Contributor onboarding audit
